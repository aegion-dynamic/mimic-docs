# Mimic Hardware Sensor Emulator - ESP8266 Test Suite

This folder contains the test sketches and documentation for validating the **Mimic** STM32-based hardware sensor emulator using an ESP8266 (NodeMCU).

## 🚀 Overview

The Mimic firmware transforms an STM32F411 (BlackPill) into a high-fidelity hardware emulator for common sensors. It uses the STM32's native hardware peripherals (SPI1 and I2C1) in **Slave Mode** to respond to requests from a Master (ESP8266) with zero-latency, realistic data.

## 🔌 Hardware Connections

### 1. BMP280 (SPI Protocol)
The BMP280 emulation uses the **Hardware SPI1** peripheral. 

| ESP8266 (NodeMCU) | STM32 BlackPill | Function |
|-------------------|-----------------|----------|
| **D5** (GPIO14)   | **PA5**         | SCK (Clock) |
| **D6** (GPIO12)   | **PA6**         | MISO (Data Out) |
| **D7** (GPIO13)   | **PA7**         | MOSI (Data In) |
| **D8** (GPIO15)   | **PA4**         | CS (Chip Select) |
| **G** (GND)       | **GND**         | Ground (Common) |

### 2. MPU6050 (I2C Protocol)
The MPU6050 emulation uses the **Hardware I2C1** peripheral.

| ESP8266 (NodeMCU) | STM32 BlackPill | Function |
|-------------------|-----------------|----------|
| **D1** (GPIO5)    | **PB6**         | SCL (Clock) |
| **D2** (GPIO4)    | **PB7**         | SDA (Data) |
| **G** (GND)       | **GND**         | Ground (Common) |

> [!IMPORTANT]
> **Common Ground**: You MUST connect the GND of the ESP8266 to the GND of the STM32, otherwise communication will fail.
> **Pull-ups**: I2C requires pull-up resistors. While the STM32 uses internal 40k pull-ups, for longer wires, adding external 4.7k resistors is recommended.

## 🛠️ Software Setup

### Prerequisites
- `arduino-cli` installed in `Esp-Test/bin`.
- ESP8266 board support installed.
- Required Libraries:
  - `Adafruit BMP280 Library`
  - `Adafruit MPU6050 Library`
  - `Adafruit Unified Sensor`

### Running the Emulation
1. **Flash the STM32** with the latest Mimic firmware.
2. **Launch the CLI** on your PC:
   ```bash
   # For BMP280 (SPI)
   sudo mimic bmp280 --protocol spi
   
   # For MPU6050 (I2C)
   sudo mimic mpu6050 --protocol i2c
   ```
3. **Upload the Test Sketch** to the ESP8266:
   ```bash
   ./bin/arduino-cli compile --fqbn esp8266:esp8266:nodemcuv2 MPU6050.ino
   ./bin/arduino-cli upload -p /dev/ttyUSB0 --fqbn esp8266:esp8266:nodemcuv2 MPU6050.ino
   ```

## ⚠️ Limitations & Notes

- **SPI Latency**: The STM32 uses a "Chambered" response strategy where it pre-loads the Chip ID into the hardware register to eliminate software latency. However, SPI speeds above 1MHz may require DMA for stable burst reads.
- **I2C Address**: The MPU6050 uses address `0x68`. The BMP280 uses address `0x77` (or `0xD0` shifted).
- **Simulation Logic**: The actual sensor values are generated by the Python script on your PC and sent to the STM32 via UART. The STM32 serves as the real-time hardware bridge.
- **Protocol Switching**: The STM32 can handle both I2C and SPI, but they must be initialized via the CLI before the ESP8266 attempts to connect.
